# Maintenance Roadmap

## Purpose

This file records short-term maintenance direction for the project while keeping it small, deterministic, and dependency-free.

## Maintenance Principles

* Preserve deterministic execution behavior.
* Keep the runtime dependency-free.
* Keep examples small and reproducible.
* Keep CI fast and offline.
* Avoid production credentials, private paths, or external services.

## Near-Term Roadmap

* Improve schema validation coverage.
* Expand adapter documentation for future implementers.
* Add more negative-case tests for invalid manifests.
* Keep release notes synchronized with public-facing changes.
* Improve reviewer-facing documentation clarity.

## Non-Goals

* No real AI inference.
* No cloud runtime integration.
* No production payment, licensing, or credential systems.
* No dependency-heavy runtime expansion.
