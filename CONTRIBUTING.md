# Contributing

Contributions are welcome. This is a small, focused open-source project.

## Guidelines

1. **Keep it simple** — zero dependencies, standard library only.
2. **No commercial logic** — no licensing, payment, order, or authentication code.
3. **Tests required** — all contributions must include or update tests.
4. **Python 3.10+** — use standard type hints and rom __future__ import annotations.

   python -m unittest discover -s tests

## Pull request process

1. Fork the repository and create a feature branch.
2. Make your changes with clear commit messages.
3. Ensure all tests pass.
4. Open a pull request describing the change.

## Code of conduct

Be respectful, constructive, and inclusive. This is a hobby open-source project — everyone is here to learn and share.
